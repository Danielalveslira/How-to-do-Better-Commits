## Many times GitHub is confused with Git.  
and I understand that well, because the names look a little similar, lol.

But actually, the two are very different. Let’s understand this with an analogy.

Confusing Git and GitHub is like confusing a video file on your PC with YouTube.

Just like YouTube hosts `.mp4` videos, it lets you host, share, and distribute videos. It also works like a social network, where people can comment on videos, like them, and more.

Following the same idea, GitHub is like YouTube, but for Git files. On GitHub, you can host Git repositories, share them, distribute them, and it also has social features like pull requests and much more.

And in the same way that on YouTube you have an account to host your videos, on GitHub we also create an account to host our Git repositories. **The beginning of a big journey!**

When you create your GitHub account with your *username*, people can access it through the address `github.com/username`, or mention you with `@username`.

I recommend choosing a good name for your GitHub account, because when you create your username, even though it is possible to change it later, many references that point to your repositories, your profile, and other very important things can break. 🤝

## Creating a New Repository

A repository on GitHub is where everything begins. From it, we can create projects, share code, and even publish websites. Now I will show you how to create a repository from scratch.

On GitHub’s home page, in this layout 
![image](https://i.imgur.com/W9JaCgq.png) many things can change over time. So, more important than memorizing the interface is understanding the main parts that usually stay the same. And those are the parts I will highlight here.

By clicking the **New** button or using the menu in the top right corner, we can find 3 main icons: the bell, which shows notifications about projects we are working on; the **+** sign, where we create a new repository; and the profile icon, which shows account options like **Settings**.

When we click the **+** icon, we go to this screen 
![image](https://i.imgur.com/PPffVPt.png)

In **Repository name**, we enter the name of the repository. It is not possible to create a name that already exists in the wrong context, and depending on the case GitHub may show an error 
![image](https://i.imgur.com/1H3pIS4.png).

Now comes an important part: **what name should you choose?**  
The nice thing is that GitHub itself gives us a good tip right below 
![image](https://i.imgur.com/WYMJoFP.png)

> "Great repository names are short and memorable."

And I totally agree. Choosing a short and memorable name is very important. It helps people understand the project better, and it can also help people find it more easily.

So: **short and memorable names** 🤝

For now, I will use `test`.

Another thing that also helps explain your project is the **description** field. You can fill it with a short summary of what the repository does, and this description appears highlighted on the repository page **in about section**
![image](https://i.imgur.com/HuwbNmP.png)

You can also choose if the repository will be **public** or **private**:

- **Public**: anyone can view it.
- **Private**: only you and people with permission can access it.

An interesting thing about a private repository is that if someone from outside tries to access it, GitHub does not show its content. In practice, the person will see a page as if that repository does not even exist 

To show this, I have a private repository. So I open an incognito tab, search for it, and look at this 
![image](https://i.imgur.com/bV1DRQq.png)  
It returns **404**, as if it does not exist.

Very cool!!

The next and last steps are deciding if you want to create the repository with some starting files, such as:

- `README.md`
- `.gitignore`
- `LICENSE`

These files can be very useful at the beginning of a project.

Well, there are still many valuable things we can learn from this, but now I will leave that part to you: search more about what `.gitignore`, `LICENSE`, and the other files mentioned here are, and what they are used for. 🤝💪
